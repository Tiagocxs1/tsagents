# Template: Estrutura da Landing Page de Vendas

Use no **Passo 3** do workflow para propor a estrutura e pedir aprovação. A ordem **não é fixa**:
ela se adapta ao nível de consciência (`reference/01`). Escolha um dos 3 presets e ajuste.

Para cada seção proposta, escreva uma linha: **[Seção] → objetivo persuasivo → lead/framework/
gatilho que vai usar.** Só depois de aprovado, escreva a copy.

---

## O menu completo de seções (o "lego")
1. Faixa de topo (eyebrow): scent do anúncio + identificação do avatar.
2. Headline: a Big Idea / promessa (`reference/03`).
3. Subheadline: amplia + tease do mecanismo.
4. Hero: CTA + prova visual (mockup/foto/VSL).
5. Barra de credibilidade: números, logos, "como visto em".
6. Problema + agitação (PAS).
7. A virada / nova oportunidade (por que o método antigo falha).
8. Solução + mecanismo único.
9. Apresentação do produto (o que é, como funciona, o que recebe).
10. Benefícios (bullets, benefício ligado a Life Force).
11. Provas (depoimentos, casos, dados, autoridade).
12. Oferta + value stack (Grand Slam Offer).
13. Bônus.
14. Preço + ancoragem e justificativa.
15. Garantia / reversão de risco.
16. Urgência / escassez (real).
17. FAQ / quebra de objeções.
18. CTA final + recap.
19. P.S. de fechamento.

---

## Preset A: Página LONGA (tráfego frio, unaware / problem-aware)
Mais degraus a subir. Abre por história ou problema (lead indireto). Estrutura base PASTOR.
`Eyebrow → Headline (problema/curiosidade) → Subhead → Hero → Barra de credibilidade →
Problema+Agitação → A virada → História/Mecanismo (Epiphany Bridge) → Apresentação do produto →
Benefícios → Provas → Oferta+stack → Bônus → Preço+justificativa → Garantia → Urgência → FAQ →
CTA final → P.S.`
- Considere preceder de um **advertorial** (`templates/advertorial-structure.md`).

## Preset B: Página MÉDIA (solution-aware / product-aware)
Já sabe que existe solução. Abre por promessa + mecanismo (lead direto/semi). Base AIDA.
`Eyebrow → Headline (promessa+mecanismo) → Subhead → Hero (com CTA) → Barra de credibilidade →
Problema (curto) → Solução + mecanismo único → Benefícios → Provas → Oferta+stack → Bônus →
Garantia → Preço → Urgência → FAQ → CTA final`

## Preset C: Página CURTA (most-aware / quente / retargeting)
Quer comprar, falta o empurrão. Abre pela oferta (lead de oferta).
`Headline (oferta) → Subhead → Hero (CTA) → Benefícios-chave → Prova rápida → Oferta+stack →
Garantia → Urgência → CTA final`

---

## Como adaptar (cruze com o diagnóstico do Passo 2)
- **Consciência** decide o preset e o lead (`reference/01`, `reference/02`).
- **Sofisticação** decide o ângulo da headline: promessa (1-2), mecanismo (3-4), identificação (5).
- **Oferta fraca** no diagnóstico? Reforce as seções de prova, value stack, garantia e bônus, e
  avise o usuário (`reference/05`).
- **Nicho sensível** (saúde/dinheiro)? Mais educação e mecanismo, claims conservadores
  (`reference/09`), provavelmente advertorial antes.
- **Serviço (não produto):** dê mais peso a autoridade, casos/resultados, processo de trabalho e
  garantia/risco; a "apresentação do produto" vira "como funciona o serviço".

## Formato de saída do Passo 3 (proposta de estrutura)
Apresente assim, e PARE para aprovação:

```
Diagnóstico: [consciência] · [sofisticação] · lead [tipo] · Big Idea: "[...]"
Preset sugerido: [A/B/C], porque [motivo].

1. [Seção] → [objetivo] → [lead/framework/gatilho]
2. ...
(seguir com todas as seções na ordem)

Aprova essa estrutura ou quer ajustar antes de eu escrever a copy?
```
