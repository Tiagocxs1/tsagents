## Arquitetura de Sistemas de Movimento

### Motion Design Tokens (Sistema Global)

```
DURAÇÃO:     fast(100ms) | normal(200ms) | slow(350ms) | expressive(500ms)
EASING:     standard | decelerate | accelerate | spring{stiffness, damping, mass}
ESCALA:     micro(componente) | meso(módulo) | macro(página/cena) | mega(campanha)
INTENSIDADE: subtle | moderate | expressive | dramatic
```

### Spatial Intelligence

O movimento não existe no vácuo — ele define e respeita o espaço.

- **Z-Axis Depth:** Elevation, shadow, parallax, layering
- **Flow Direction:** Left-to-right (progresso), up (crescimento), down (revelação), center (foco)
- **Spatial Memory:** Elementos preservam posição lógica em transições
- **Physics-Based Motion:** Massa, inércia, atrito, resiliência (spring physics)

### Timing & Rhythm

| Ritmo | Sensação | Uso |
|-------|----------|-----|
| Allegro (100-200ms) | Rápido, responsivo | Micro-interações, feedback tátil |
| Moderato (200-400ms) | Natural, confortável | Transições de tela, navegação |
| Largo (400-800ms) | Dramático, cinematográfico | Hero animations, revelações |
| Legato (encadeado) | Fluido, contínuo | Scroll-driven, parallax |

### Easing System

| Easing | Fórmula | Uso |
|--------|---------|-----|
| Standard | `cubic-bezier(0.4, 0.0, 0.2, 1)` | Transições gerais (Material Design) |
| Decelerate | `cubic-bezier(0.0, 0.0, 0.2, 1)` | Elementos entrando (de fora) |
| Accelerate | `cubic-bezier(0.4, 0.0, 1, 1)` | Elementos saindo |
| Spring | `spring(stiffness: 100, damping: 10)` | Natural, físico, iOS-like |
| Bounce | `cubic-bezier(0.68, -0.55, 0.27, 1.55)` | Expressivo, lúdico (usar com moderação) |

## Cinematic UX

### Princípios de Movimento Cinematográfico

1. **Establishing Shot:** Primeiro movimento define o espaço e contexto
2. **Continuity Editing:** Transições mantêm coerência espacial (match cut, cross dissolve)
3. **Montage:** Sequência rítmica de movimentos para compressão de tempo
4. **B-Roll Integration:** Movimento secundário que apoia a narrativa principal
5. **Depth of Field Motion:** Elementos em foco vs desfoque criam profundidade

### Câmera Virtual para UI

| Técnica | Efeito | Aplicação |
|---------|--------|-----------|
| Dolly Zoom | Tensão, revelação | Landing pages dramáticas |
| Whip Pan | Transição rápida | Entre seções |
| Tracking Shot | Continuidade | Scroll narrativo |
| Rack Focus | Mudança de atenção | Hero sections |
| Steadicam | Fluidez natural | Onboarding, tours |
| Parallax | Profundidade | Background, camadas |

### Scroll-Driven Animation

- **Reveal animations:** Elementos aparecem ao scroll com distorção temporal
- **Pin & Unpin:** Seções fixas enquanto o fundo se transforma (narrative scroll)
- **Progress-linked:** Animação atrelada à posição do scroll (0% a 100%)
- **Momentum-based:** Scroll inercial afeta a velocidade/amplitude da animação

## AI Video Motion

### Motion para Geração de Vídeo por IA

Ao criar prompts de motion para vídeo gerado por IA (Kling, Sora, Veo, Wan, Seedance):

**Sempre declarar:**
- Tipo de movimento de câmera (dolly, pan, tilt, crane, handheld)
- Direção e velocidade (slow push-in, rapid dolly out)
- Intenção emocional do movimento (íntimo, dramático, documental)
- Duração do take (contínuo vs. corte)
- Interação sujeito-câmera (olhar, quebra da quarta parede)

**Hierarquia do Prompt de Motion:**
```
[CAMERA MOVE] + [SUBJECT ACTION] + [ENVIRONMENT RESPONSE] + [LIGHT CHANGE] + [FOCUS DEPTH]
```

Exemplo: *Slow dolly forward with subject turning to camera, bokeh particles drifting across foreground, subtle focus pull from subject to background.*

### Motion Transitions para Edição

| Transição | Duração | Uso |
|-----------|---------|-----|
| Match Cut | 0.5-1s | Continuidade visual entre takes |
| Whip Pan | 0.2-0.4s | Transição rápida, energia |
| Push/Pull | 0.5-0.8s | Mudança de cena |
| Morph | 0.5-1.5s | Transformação de forma |
| Light Leak | 0.3-0.6s | Transição orgânica, analógica |
| Glitch | 0.1-0.3s | Tech/digital |
| Speed Ramp | variável | Aceleração/desaceleração dramática |

## Kinetic Typography

### Hierarquia de Movimento Tipográfico

1. **Narrativa-visual:** O texto se move com o ritmo da fala
2. **Emocional:** A forma do movimento reflete o tom (agressivo = staccato, suave = fluido)
3. **Hierárquico:** Palavras-chave se destacam com movimento próprio
4. **Temporal:** Timing sincronizado com áudio (word-by-word, phrase-by-phrase)

### Técnicas de Kinetic Typography

| Técnica | Descrição | Uso |
|---------|-----------|-----|
| Reveal | Texto aparece letra a letra | Hooks, linhas importantes |
| Scale Emphasis | Palavra-chave cresce | Destaque dramático |
| Stagger | Palavras entram em sequência | Leitura guiada |
| Float/Drift | Texto flutua suavemente | Atmosférico, poético |
| Warp/Distort | Texto se deforma | Transições, impacto |
| Mask Reveal | Texto revelado por shape | Cinemático, premium |

## Micro-Interações

### Anatomia de uma Micro-Interação

1. **Trigger:** O que inicia (hover, tap, scroll, voice)
2. **Feedback:** O que acontece (visual change, sound, haptic)
3. **Duration:** Por quanto tempo
4. **Resolution:** Como termina

### Catálogo de Micro-Interações

## Performance & Restrições

### Regras de Performance

- **60fps é padrão.** 30fps para campanhas, 120fps+ para gaming/premium
- **Micro-interações:** ≤200ms, nunca bloquear thread principal
- **Scroll-driven:** Usar `IntersectionObserver` / `ScrollTimeline` (não `scroll` events)
- **GPU Aceleration:** Animar apenas `transform` e `opacity` (nunca `left`, `top`, `width`, `height`)
- **Reduced Motion:** Respeitar `prefers-reduced-motion: reduce` com versões estáticas
- **Mobile First:** Animações complexas têm fallback para dispositivos de baixa performance

### Hierarquia de Implementação Técnica

1. CSS transitions/animations (simples, performático)
2. GSAP (complexo, timeline-driven)
3. Lottie/Bodymovin (gráficos vetoriais animados)
4. Remotion (código → vídeo programaticamente)
5. Rive (state machine, interactive motion)
6. WebGL/Three.js (3D, partículas, shaders)
